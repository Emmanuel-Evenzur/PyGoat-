Lab Instructions

This lab demonstrates a reflected XSS vulnerability. The page reflects back user input without proper escaping. Try the following:

Steps to Try:
  1. First enter a valid company name like "Facebook" to see normal behavior
  2. Try entering an HTML tag: <h4>Hello</h4>
  3. Notice how your input gets rendered as actual HTML
  4. Can you make it execute JavaScript using <script> tags?
Vulnerability Details

The vulnerability exists because user input is being reflected back with the safe filter in the template: 

<h3>The company '{{query|safe}}' You searched for is not Part of FAANG</h3>
This means any HTML or JavaScript in the input will be executed by the browser instead of being escaped.
